Build is where the design becomes real. We engineer to production quality, review
in short cycles, and keep a working preview live the whole way through.

## How we build

<Steps>
  <Step title="Setup">
    We stand up the repo, hosting, and a preview environment on day one.
  </Step>
  <Step title="Cycles">
    We build in short cycles, each ending with something you can click through.
  </Step>
  <Step title="Quality pass">
    Performance, accessibility, responsiveness, and cross-browser checks before
    launch.
  </Step>
</Steps>

## Previewing work

Every change is deployed to a preview URL before it reaches production. You can
review real work in a browser at any point — no need to wait for a scheduled
demo.

<CodeGroup>

```bash npm
npm install
npm run dev
```

```bash pnpm
pnpm install
pnpm dev
```

```bash yarn
yarn install
yarn dev
```

</CodeGroup>

<Info>
  You don't need to run anything yourself — the preview URL is always live. The
  commands above are here only if your team wants to run it locally.
</Info>
